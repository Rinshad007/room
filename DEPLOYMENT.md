# Budget Buddy Backend — Deployment Guide

This guide walks you through deploying your **Budget Buddy Backend** completely on **Render** using the provided `render.yaml` blueprint configuration.

---

## 🛠 Prerequisites

1. A **GitHub account** with this repository pushed/synced to it.
2. A free account on **[Render](https://render.com/)**.

---

## 🚀 Step-by-Step Deployment on Render

### Step 1: Push your code to GitHub
Make sure all your latest local commits (including the recent PostgreSQL/SQLite fixes) are pushed to your remote repository on GitHub:
```bash
git push origin main
```

### Step 2: Deploy using Render Blueprint
Render supports **Blueprint** deployments, which read our `render.yaml` file to automatically spin up both the FastAPI backend service and the PostgreSQL database, link them together, and configure everything.

1. Go to your **[Render Dashboard](https://dashboard.render.com/)**.
2. Click the **"New +"** button in the top right and select **"Blueprint"** (or go to **"Blueprints"** from the top menu).
3. Connect your **GitHub account** if you haven't already.
4. Select your `room` (or `budget-buddy`) repository.
5. Provide a **Group Name** (e.g., `budget-buddy-prod`).
6. Click **"Apply"**!

Render will now start provisioning:
- A new managed **PostgreSQL Database** (`budget-buddy-db`)
- A new **Web Service** (`budget-buddy-backend`)

---

### Step 3: Configure Environment Variables in Render
After the services are created, you will need to update two important environment variables on the Render Dashboard:

1. In the Render Dashboard, click on your **`budget-buddy-backend`** Web Service.
2. Go to the **Environment** tab on the left.
3. Update the following values:
   * **`SECRET_KEY`**: Replace `please-update-in-dashboard` with a secure random string (at least 32 characters, e.g., generated by running `openssl rand -hex 32` locally).
   * **`ALLOWED_ORIGINS`**: Replace `please-update-in-dashboard` with your frontend URL (e.g., your Netlify URL: `https://eloquent-kringle-38f01c.netlify.app`).

4. Click **"Save Changes"**. Render will automatically redeploy the backend with the new settings.

---

## 🔍 Database Table Creation
You **do not** need to run database migrations manually on Render! The application is configured to automatically inspect the database connection and generate all necessary tables on startup via:
```python
async with async_engine.begin() as conn:
    await conn.run_sync(Base.metadata.create_all)
```

---

## 🟢 Verification
Once the deploy status turns green (**"Live"**):
1. Visit your backend URL followed by `/health` (e.g., `https://budget-buddy-backend.onrender.com/health`). You should see:
   ```json
   {
     "status": "ok",
     "app": "Budget Buddy"
   }
   ```
2. You can access the interactive Swagger API documentation at `https://budget-buddy-backend.onrender.com/docs`.
